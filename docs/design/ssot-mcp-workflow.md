# SSOT MCP Workflow

## 1. 목적
- MCP/Copilot 작업 시 SSOT 해석 순서와 충돌 판정 기준을 고정한다.

## 2. 우선순위 (고정)
- **(헌법) basic > API > MODEL > UI > DB > _index**

## 3. 작업 절차
1. 대상 ScreenID와 문서 범위를 확정한다.
2. 충돌 여부를 우선순위 기준으로 판정한다.
3. 상위 문서 기준으로 하위 문서를 정렬한다.
4. 구현 전 `계산 결과 저장 금지`, `Soft delete`, `FX 원장 원칙` 반영 여부를 검증한다.
5. 변경 사유(충돌 근거)를 남긴다.

## 4. 금지 사항
- 문서에 없는 기능 창작 금지
- 우선순위 역전 해석 금지
- 화면 책임 혼합 금지
