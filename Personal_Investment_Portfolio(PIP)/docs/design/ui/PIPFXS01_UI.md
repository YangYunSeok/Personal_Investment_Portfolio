## 1. 화면 구성
### 1.1 레이아웃
- 상단: 화면 타이틀 `Exchange (환전)`
- 본문: FX 입력 폼 카드 1개
- 하단: Primary 버튼 `저장`, Secondary 버튼 `취소`

### 1.2 폼 필드(표시/입력)
| 구역 | 필드 | UI 컴포넌트 | 비고 |
|---|---|---|---|
| 기본 | 거래일(tradeDate) | DatePicker | 기본값: 오늘 |
| 기본 | 계좌(accountId) | Select | 계좌 단위 관리 원칙 훅 |
| 출금 | 출금통화(fromCurrency) | Select | 통화 리스트 |
| 출금 | 출금금액(fromAmount) | Number | 소수점 허용 여부는 통화 정책에 종속 |
| 입금 | 입금통화(toCurrency) | Select | from과 동일 선택 불가 |
| 입금 | 입금금액(toAmount) | Number |  |
| 환율 | 환율(fxRate) | Number | 사용자 입력값 저장(계산 금지) |
| 수수료(옵션) | 수수료 통화/금액 | Select + Number | 둘 중 하나 입력 시 나머지 필수 |
| 메모(옵션) | memo | Textarea |  |

## 2. 사용자 흐름
### 2.1 진입
- 메뉴/네비게이션에서 `Exchange` 선택
- 또는 Activity Log에서 `FX 추가`로 진입(연계는 Activity Log SSOT에 종속)

### 2.2 저장
1) 사용자가 입력 → `저장` 클릭  
2) 클라이언트 1차 검증  
3) 서버 API 호출 → 성공 시 목록(또는 직전 화면)으로 이동

### 2.3 취소
- 변경사항이 있을 때: “저장하지 않고 나가시겠습니까?” 확인 모달(권장)
- 변경사항이 없을 때: 즉시 뒤로가기

## 3. 에러/검증 메시지
- 필수값 누락: 각 필드 하단에 인라인 메시지
- 통화 동일 선택: `출금 통화와 입금 통화는 달라야 합니다.`
- 금액/환율 0 이하: `0보다 큰 값을 입력하세요.`
- 서버 오류: 토스트 + 폼 상단 공통 에러 영역(권장)

## 4. 비기능(UI 규칙)
- 입력값만 저장하며 계산값(환차손익 등)을 화면에서 “확정 값”처럼 보여주지 않는다. fileciteturn0file0
- 금액 포맷은 통화별 소수점 정책을 따르며(예: KRW 0자리), 정책은 공통 SSOT에 종속.
